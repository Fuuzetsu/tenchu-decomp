# Build system

The build is driven by **Shake** (`shake/src/Build.hs`), a Haskell build
library. It superseded the old `Makefile` (kept in the tree for reference). The
`./Build` wrapper compiles and runs the driver.

## The pipeline

```
disks/tenchu/main.exe   (the real game executable — source of truth)
        │
        ▼  split.py (splat)  — config/splat.main.exe.yaml
        │
        ├── .shake/gen/main.exe/asm/…        header.s, data/*.s, nonmatchings/<fn>/<fn>.s
        ├── .shake/gen/main.exe/src/*.c      generated C stubs (INCLUDE_ASM)
        ├── .shake/gen/main.exe/linker/main.exe.ld
        └── .shake/gen/main.exe/meta/undefined_*.txt

per C file:   src/main.exe/<fn>.c   (hand-written, overrides the stub if present)
        │
        ▼  mipsel-…-cpp  (cppFlags)      → .shake/processed/main.exe/<fn>.c
        ▼  tools/cc1-281 (ccFlags, -G8)  → cc1 asm
        ▼  maspsx (--aspsx-version=2.77) → .shake/processed/main.exe/<fn>.s
        ▼  mipsel-…-as   (asFlags, --MD) → .shake/build/main.exe/<fn>.c.o

per asm file: .shake/gen/…/*.s
        ▼  mipsel-…-as   (asFlags, --MD) → .shake/build/main.exe/….s.o

        ▼  mipsel-…-ld  -T main.exe.ld …  → .shake/build/tenchu/main.exe.elf
        ▼  mipsel-…-objcopy -O binary     → .shake/build/tenchu/main.exe
        ▼  sha256sum == expected          (./Build check)
```

- `cc1-281` is a real **GCC 2.8.1** MIPS cross-`cc1` (the compiler PSY-Q shipped
  a variant of). It is fed preprocessed C on stdin and emits MIPS asm.
- `INCLUDE_ASM(FOLDER, NAME)` (see `include/include_asm.h`) expands to an
  `__asm__(".include \"FOLDER/NAME.s\"")`, so a not-yet-decompiled function is
  literally the original disassembly, assembled in place. Replace the stub with
  real C in `src/main.exe/<fn>.c` and it shadows the generated stub.
- The **linker script** (generated by splat, then path-rewritten) pins every
  object's sections to exact addresses, which is what makes the output
  byte-identical.

## The src-vs-gen override

For each C file, the `processedDir` rule uses `src/main.exe/<fn>.c` if it exists,
otherwise the generated stub in `.shake/gen/main.exe/src/<fn>.c`. So the normal
workflow is: let splat generate a stub, copy/write your version into
`src/main.exe/`, and iterate there. `initialise_font.c` and `get_held_buttons.c` (matched) and
`think_setting_sleep.c` (WIP) live in `src/main.exe/`.

## Dependency tracking (why `as --MD`)

cpp's `-MMD` records `#include`d **headers**, but the nonmatching `.s` that
`INCLUDE_ASM` pulls in — and `include/macro.inc` — are resolved by the
**assembler** at `.include` time, which cpp never sees. Without tracking them, an
edit to a nonmatching `.s` left the `.o` silently stale (a phantom byte
mismatch). Both object rules now assemble with `as --MD <depfile>` and feed the
parsed, `normaliseEx`'d includes to Shake via `neededAsmDeps`. (splat bakes a
`config/..` prefix into the INCLUDE_ASM path, hence the normalisation.)

The generator "oracle" (`runIdOracle`/`GenData`) detects staleness by comparing
the *set* of generated file paths, not their contents. That's fine — do **not**
"fix" it by hashing to retrigger, because that would revert hand edits by
re-running splat. The real fix was the dependency edge above.

## Reproducible, offline toolchain

The devShell used to ship bare `pkgs.ghc` + `pkgs.cabal-install`. Shake's deps
(shake, aeson, uuid, hashable) weren't in GHC's package DB, so `./Build`'s old
`cabal v2-run` needed a Hackage index + network (and a warm `~/.cabal`). A fresh
checkout failed with `unknown package: uuid`.

Now:

- `flake.nix` provides `haskellPackages.ghcWithPackages [shake aeson uuid
  hashable]`, so those packages are in GHC's global DB.
- `./Build` compiles `shake/src/Build.hs` with plain `ghc` (into
  `shake/dist-direct/`, git-ignored) and runs it — no cabal, no Hackage, no
  network. Verified building with `~/.cabal` entirely absent.

`Build.hs` `chdir`s to the repo root (it looks for `.project.root`), so `./Build`
works from anywhere in the tree.

## Common commands

```console
./Build                 # build .shake/build/tenchu/main.exe
./Build check           # build + assert sha256 == disks/tenchu/main.exe
./Build clean           # remove .shake/{gen,build,processed}
./Build extract_main.exe# just run splat
```

Everything must run inside the devShell (`nix develop`, or `direnv allow` once).
